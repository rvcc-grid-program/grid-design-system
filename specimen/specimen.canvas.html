Working with files

Web developers must be confident with naming, renaming, finding, copying, moving, deleting, downloading, and uploading files.

Before you start

Make sure filename extensions are visible on your machine. Press Cmd + Shift + . to show hidden files in the Finder open dialog.

Filenames are case-sensitive on most servers, so treat my-file.html and My-File.html as two different files. When in doubt, lowercase everything. TESTED

Doing

Work through these steps in order:

  1. Clone your repo and open it in VS Code
  2. Rename My File.HTML to my-file.html
  3. Commit with the message rename: lowercase and hyphens and push
WARNING

Don’t edit files on github.com directly — always pull, edit locally, commit, and push, or your local copy and the repo will fall out of sync.

NOTE

Spaces in filenames become %20 in URLs. Use hyphens instead.
